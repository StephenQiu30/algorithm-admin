## ADDED Requirements

### Requirement: Post management SHALL support complete review-oriented operations
The system SHALL make the existing post management page sufficient for routine review work within the current admin design.

#### Scenario: Administrator focuses on pending review items
- **WHEN** an administrator enters the existing post management page
- **THEN** the page SHALL allow the administrator to quickly identify and focus on posts that require review

#### Scenario: Administrator performs a review decision
- **WHEN** an administrator reviews a post in the existing post management workflow
- **THEN** the page SHALL present the information needed to complete that review action without requiring a new page structure

### Requirement: Comment management SHALL provide operational context
The system SHALL make the existing comment management page sufficient for administrators to understand and handle comments in context.

#### Scenario: Administrator inspects a comment
- **WHEN** an administrator views a comment in the existing comment management page
- **THEN** the page SHALL show enough related context to understand which content and which user the comment belongs to

#### Scenario: Administrator needs to continue handling related content
- **WHEN** an administrator determines that the related post needs follow-up
- **THEN** the existing management flow SHALL provide a direct way to continue to the related content handling path
